# Introduction

Greenmask is an open-source util for managing logical backups and perform dump obfuscation. It is stateless util (is not
required any database schema changes), has a variety of storages and provide comprehensive obfuscation features. Was
designed as easy customizable and backward compatible with PostgreSQL utils.

# When and why did Greenmask appear?

According to the real business demand need for an obfuscation system that would be easily accessible, the team started
to explore current solutions. They are good for getting started, but we found that integrating them into the staging
environment is quite difficult and most of them are not working in a declarative approach, and it is difficult to
extend them. Since that point, we started to explore the suitable way to implement it and became to the Greenmask
implementation.

# Our purpose

Grenmask util is going to be a core system within the Greenmask environment we are trying to develop a comprehensive
UI-based solution for managing obfuscation procedures. We understand that it is difficult to maintain an obfuscation
state during the software lifecycle and Greenmask trying to provide useful tools and features for keeping 
obfuscation fresh procedure, easy to develop and easy deliver the solution.
